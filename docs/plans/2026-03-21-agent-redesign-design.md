# Novel Agent ReAct 架构重新设计

> 将小说创作系统从线性状态机模式重构为类似 Claude Code 的完整 Agent 模式

---

## 一、设计背景

### 当前问题

现有架构是**线性状态机模式**:
```
用户输入topic → LLM生成 → 保存章节 → 更新记忆 → 循环
```

**问题**:
- 流程固定,缺乏灵活性
- 无法根据实际情况动态调整策略
- 质检失败后无法自动规划修订流程

### 目标

构建**完全自主的 Agent 模式**,类似 Claude Code:
- Agent 自行规划整个章节创作流程
- 动态决策,根据观察结果调整行动
- 支持并行执行子任务
- 用户只提供高层指令

## 三、核心组件设计

### 3.1 Tool Registry (4个核心工具)

| 工具 | 上下文 | 用途 | 示例 |
|------|--------|------|------|
| `bash` | N/A | 执行任意shell命令,文件操作 | `cat project.json`, `ls chapters/` |
| `generate_content` | 共享 | LLM生成正文/大纲/摘要 | "根据大纲生成第10章" |
| `dispatch_agent` | 独立 | 派发SubAgent执行子任务 | 质检、记忆更新 |
| `ask_user` | N/A | 向用户提问 | "角色A的行为是否合理?" |

**设计原则**:
- 能用bash做的事情就用bash做
- 需要 LLM 理解/创作的任务用 `generate_content`
- 需要用户决策时用 `ask_user`
- 可并行执行的子任务用 `dispatch_agent`

### 3.2 SubAgent 类型 (独立上下文)

| SubAgent | 职责 |
|----------|------|
| `quality_check` | 质量检查 (一致性/逻辑/风格) |
| `memory_update` | 从生成内容中提取并更新记忆 |
| `outline_gen` | 生成章节大纲 |

**上下文策略 (混合模式)**:
- **正文生成**: 主 Agent 直接调用 `generate_content`,共享上下文
- **质检/记忆等**: 通过 `dispatch_agent` 派发独立 SubAgent,隔离上下文

### 3.3 ReAct 循环

```
┌─────────────────────────────────────────────────────────────┐
│                      ReAct Loop                             │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 1. THOUGHT (思考)                                   │   │
│  │    "用户想写第10章,我需要先检查项目状态..."          │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 2. ACTION (行动)                                    │   │
│  │    bash: "cat project.json"                         │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 3. OBSERVATION (观察)                               │   │
│  │    "项目有9章,大纲指向第10章是决战场景..."           │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 4. THOUGHT: "需要获取更多上下文,并行执行记忆更新    │   │
│  │    和读取第9章..."                                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 5. ACTION (并行)                                    │   │
│  │    dispatch_agent(type="memory_update")             │   │
│  │    bash: "cat chapters/chapter_009.md"              │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 6. OBSERVATION: "记忆已更新,第9章内容已读取..."     │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 7. ACTION: generate_content(...)                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 8. OBSERVATION: "正文生成完成,3000字"               │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 9. ACTION: dispatch_agent(type="quality_check")     │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 10. OBSERVATION: "质检通过,无问题"                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 11. ACTION: bash: "保存章节到 chapters/"            │   │
│  └─────────────────────────────────────────────────────┘   │
│                          ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 12. FINAL RESPONSE                                  │   │
│  │     "第10章已完成并保存,字数3000,无质量问题"         │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

## 五、数据流与状态管理

```

```

```json
{
  "messages": [
    { "role": "system", "content": "你是小说创作Agent..." },
    { "role": "user", "content": "写第10章" },
    { "role": "assistant", "content": "让我先检查项目状态...", "tool_calls": [...] },
    { "role": "tool", "content": "项目有9章...", "tool_call_id": "..." }

```

```
projects/<slug>/.agent-state.json
```

```json
{
  "last_action": "generate_content",
  "pending_tasks": [],
  "context_snapshot": {
    "current_chapter": 10,
```

```
用户: Ctrl+C 中断

```

## 六、关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 自主程度 | 完全自主 | 类似 Claude Code 独立完成任务 |
| 交互方式 | 对话式 | 多轮对话,上下文延续 |
| 工具数量 | 4个核心工具 | 精简,能用bash的用bash |
| 上下文策略 | 混合模式 | 正文共享,SubAgent独立 |
| 并行执行 | dispatch_agent | 质检和记忆可并行 |
